# 세션 023 — 강제 업데이트 기능의 설정 모달 이동 (v5.31)

**날짜**: 2026-04-26  
**작업자**: Gemini CLI

---

## 주요 작업 목록

### v5.31 — UI 정돈 및 설정 기능 강화
- **홈 화면 UI 정돈**: 홈 화면 하단 푸터의 버전 배지에서 `(force update)` 링크와 클릭 기능을 제거하여 시각적 노이즈를 줄임. 버전 정보만 단순 텍스트로 표시함.
- **기능 이동**: 기존의 강제 업데이트 기능을 **설정(⚙️)** 모달 내부로 이동시켜 관리적 성격에 맞는 위치로 재배치함.
- **설정 모달 개선**: '앱 정보 및 업데이트' 섹션을 신설하여 현재 버전 확인 및 강제 업데이트를 한곳에서 처리할 수 있도록 함.

---

## 변경 사항 상세

1. **js/app.js**
   - `renderHome()`: 푸터 레이아웃 수정 및 설정 모달 내 업데이트 섹션 추가.
   - `bindEvents()`: `btn-version` 리스너를 제거하고 `btn-force-update` 리스너 추가.
   - `APP_VERSION`을 `v5.30`에서 `v5.31`로 업데이트.

2. **sw.js**
   - `CACHE` 버전을 `hanja-v53`에서 `hanja-v54`로 업데이트.

3. **PROJECT.md**
   - 현재 버전 정보 갱신.

---

## 설계 결정사항

- **UI 우선순위**: 일반 사용자에게 노출될 필요가 적은 관리용 기능(강제 업데이트)은 설정 메뉴로 숨기고, 홈 화면은 학습에만 집중할 수 있는 환경으로 유지함.
