# 세션 032 — 무한 복습(Endless Review) 모드 추가 (v5.41)

**날짜**: 2026-04-26  
**작업자**: Gemini CLI

---

## 주요 작업 목록

### v5.41 — 실시간 기록형 무한 복습 시스템 도입
- **무한 복습 모드 신설**: 세션 제한(10장) 없이 사용자가 종료할 때까지 계속되는 '무한 복습' 기능을 추가함.
  - **대상**: 1회 이상 학습하여 기록에 등록된 모든 한자.
  - **우선순위**: 사용자 정의 A(오답) → B(복습 주기) → D(대기) 순으로 큐 자동 무한 보충.
- **실시간 데이터 바인딩**: 10문제가 끝나야 저장되던 기존 방식과 달리, 한 문제를 풀 때마다 즉시 `SM2` 학습 데이터와 정답률을 업데이트함.
- **UI 최적화**: 
  - 홈 화면에 **[🔄 무한 복습]** 버튼 추가 (녹색 계열로 차별화).
  - 무한 복습 시 상단 헤더에 '진행률(1/10)' 대신 '**누적 획득 XP**'를 실시간 표시하여 성취감 고취.

---

## 변경 사항 상세

1. **js/app.js**
   - `renderHome()`: 무한 복습 버튼 UI 추가.
   - `buildInfiniteQueue()`, `startInfiniteReview()`: 무한 모드 전용 큐 및 시작 로직 구현.
   - `answer()`: 무한 모드일 때 세션 종료 화면을 생략하고 큐를 자동 리필하도록 수정.
   - `APP_VERSION`을 `v5.40`에서 `v5.41`로 업데이트.

2. **sw.js**
   - `CACHE` 버전을 `hanja-v63`에서 `hanja-v64`로 업데이트.

3. **PROJECT.md**
   - 현재 버전 및 '무한 복습' 기능 정의 추가.

## 설계 결정사항

- **모드별 차별화**: [학습 시작]은 신규 한자를 포함한 정석 세션 방식을 유지하고, [무한 복습]은 이미 아는 한자를 완벽히 체득하기 위한 끝없는 트레이닝 환경으로 설계함.
- **SRS 원칙 고수**: 무한 모드에서 동일 한자를 여러 번 맞혀도 복습 주기는 하루에 한 번만 늘어나도록 설계하여 SRS 알고리즘의 무결성을 유지함.
