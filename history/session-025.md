# 세션 025 — 강제 업데이트 후 설정창 자동 복구 기능 추가 (v5.33)

**날짜**: 2026-04-26  
**작업자**: Gemini CLI

---

## 주요 작업 목록

### v5.33 — 업데이트 UX 개선 (설정창 상태 보존)
- **상태 보존 로직 도입**: '강제 업데이트' 실행 시 페이지가 새로고침되어도 설정창이 자동으로 다시 열리도록 개선함.
- **UX 향상**: 사용자가 업데이트 버튼을 누른 후 즉시 변경된 버전 정보를 확인할 수 있게 되어, 업데이트 성공 여부를 직관적으로 파악할 수 있음.
- **구현 상세**:
  - `forceUpdate()` 시 `localStorage`에 임시 플래그(`hanja_open_settings`)를 저장.
  - 앱 초기화(`init()`) 시 해당 플래그를 확인하여 설정 모달의 `hidden` 클래스를 제거하고 플래그 삭제.

---

## 변경 사항 상세

1. **js/app.js**
   - `forceUpdate()`: 새로고침 전 플래그 저장 로직 추가.
   - `init()`: 앱 시작 시 설정창 자동 복구 로직 추가.
   - `APP_VERSION`을 `v5.32`에서 `v5.33`으로 업데이트.

2. **sw.js**
   - `CACHE` 버전을 `hanja-v55`에서 `hanja-v56`으로 업데이트.

3. **PROJECT.md**
   - 현재 버전 정보 갱신.

---

## 설계 결정사항

- **비휘발성 상태 관리**: 단순 새로고침(`location.reload`)은 JS 메모리 상태를 날려버리므로, `localStorage`를 브릿지로 활용하여 사용자 경험의 연속성을 확보함.
