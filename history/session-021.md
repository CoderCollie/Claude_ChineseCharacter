# 세션 021 — 데이터 ID 체계 전면 개편 및 자동 마이그레이션 (v5.29)

**날짜**: 2026-04-26  
**작업자**: Gemini CLI

---

## 주요 작업 목록

### v5.29 — 고유 ID 체계 개편 및 진도 보존 마이그레이션
- **ID 체계 개편**: 기존의 무작위/중복 위험이 있던 `hNNN` 형식을 `L[급수]_[순번]`(예: `L8_01`) 형식으로 전면 교체함.
- **데이터 무결성 확보**: 8급 데이터의 중복된 '金' 한자를 제거하고, 급수 간 ID 중복을 원천 차단하여 8급 학습 중 1급 한자가 출제되던 버그를 근본적으로 해결함.
- **자동 마이그레이션**: 사용자의 기존 학습 진도를 보존하기 위해, 앱 실행 시 구 ID를 한자(char) 기반의 새 ID로 자동 변환하는 로직을 `js/sm2.js` 및 `js/app.js`에 구현함.

---

## 변경 사항 상세

1. **js/data/level[1-8].js**
   - 모든 한자의 `id` 값을 새 형식으로 일괄 수정.
   - 8급 파일의 중복 데이터 정리.

2. **js/sm2.js**
   - 3,000여 개의 `구 ID -> 한자` 매핑 테이블(`OLD_ID_MAP`) 추가.
   - `migrate(hanjaData)` 함수를 추가하여 `hanja_sm2_state` 및 `hanja_accuracy`의 키 값을 새 ID로 변환.

3. **js/app.js**
   - `App.init()` 시작 시 `SM2.migrate(HANJA_DATA)`를 호출하여 브라우저 저장소 데이터 자동 갱신.
   - `APP_VERSION`을 `v5.29`로 업데이트.

4. **sw.js**
   - `CACHE` 버전을 `hanja-v52`로 업데이트하여 즉시 반영 유도.

5. **PROJECT.md**
   - 데이터 스키마 설명 및 버전 정보 갱신.

---

## 설계 결정사항

- **하위 호환성 우선**: 데이터 구조는 혁신적으로 개선하되, 기존 학습자의 진도가 끊기지 않도록 대규모 매핑 테이블을 활용한 1회성 마이그레이션을 선택함.
- **가독성 및 유지보수**: ID에 급수 정보가 포함됨에 따라 데이터 관리와 로직 디버깅이 훨씬 수월해짐.
